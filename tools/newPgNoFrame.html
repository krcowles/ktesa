This 'new page / no frame' html file, with its accompanying javascript, will dynamically load and size user-selected images to fit into a row of specified height (the default for height, if not changed by the optimization routne, is 260px - if a different "seed" height is desired, change the var 'rowHeight' near the top of the javascript file). The images will fill a row of 'rowLineWidth (specified in the accompanying CSS file 'body' element - it is best to leave this as is to ensure consistency across sites). The specification of photo images begins immediately after the 'p id="page_title"'. FIRST: specify the Flickr "name" of the photo(s) you wish to be displayed, into the set of consecutive "li" elements (ul id="photos"). NOTE: the names must be an EXACT MATCH, case sensitive, to the name appearing in the Flickr ablum.. SECOND:specify the .tsv file created for this type of page (use 'mkpgfile', not 'mkgpsv_r6') corresponding to the Flickr photos in the last step. THIRD:if you wish to have other images on the page (with captions) indicate the number of images in the p id="noOfOthr". If none, specify 0 there. FOURTH:specify the full .jpg file names for any other images (not photos) to be used for which you wish to supply captions in the consecutive "li" elements (ul id="othrImg"). These may include generic map images, or other images pertinent to the hike. The images will behave as do the photos, with "popup" captions when a mouseover event occurs. The images must be available in the "../images" directory of the project. They will not be linked to the Flickr album. The text will be supplied in the next step. FIFTH:provide the text for any non-photo images in step 2, above, in the same order as the images are listed. The text should be supplied in the ul list with id="imgCaptions".SIXTH:if there are any non-captioned images, specify them in in consecutive "li" elements (ul id="nonCaps"). Use the full file spec (i.e. include ".jpg"). They too must be available in the "../images" directory. That's it - save and execute! A text file will be created which can be cut and pasted into the page template. The template CSS file will center the images and iframe on the page, provide pop-up descriptions, and links to the album.
